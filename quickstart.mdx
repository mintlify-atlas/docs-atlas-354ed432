Get nvim-treesitter up and running in your Neovim setup quickly with this guide.

## Prerequisites

Before installing nvim-treesitter, ensure you have:

- **Neovim 0.11.0 or later** (nightly build recommended)
- `tar` and `curl` available in your PATH
- [`tree-sitter-cli`](https://github.com/tree-sitter/tree-sitter/blob/master/crates/cli/README.md) version 0.26.1 or later
- A C compiler in your PATH

<Warning>
nvim-treesitter only supports the latest stable release and latest nightly prerelease of Neovim. Other versions may work but are not tested or officially supported.
</Warning>

## Installation

<Steps>
  <Step title="Install the plugin">
    Install nvim-treesitter using your preferred package manager:

    <CodeGroup>
    ```lua lazy.nvim
    {
      'nvim-treesitter/nvim-treesitter',
      lazy = false,
      build = ':TSUpdate'
    }
    ```

    ```lua packer.nvim
    use {
      'nvim-treesitter/nvim-treesitter',
      run = ':TSUpdate'
    }
    ```

    ```lua rocks.nvim
    :Rocks install nvim-treesitter
    ```

    ```vim vim-plug
    Plug 'nvim-treesitter/nvim-treesitter', {'do': ':TSUpdate'}
    ```
    </CodeGroup>

    <Note>
    This plugin does **not** support lazy-loading. Set `lazy = false` in lazy.nvim or load it immediately.
    </Note>
  </Step>

  <Step title="Install language parsers">
    Install parsers for the languages you work with:

    ```vim
    :TSInstall lua python javascript typescript rust
    ```

    Or programmatically in your `init.lua`:

    ```lua
    require('nvim-treesitter').install({ 'lua', 'python', 'javascript', 'typescript', 'rust' })
    ```

    <Tip>
    The `:TSUpdate` command in the build step will automatically update all installed parsers when you update the plugin.
    </Tip>
  </Step>

  <Step title="Enable syntax highlighting">
    To enable tree-sitter syntax highlighting for your languages, add this to your `init.lua`:

    ```lua
    vim.api.nvim_create_autocmd('FileType', {
      pattern = { 'lua', 'python', 'javascript', 'typescript', 'rust' },
      callback = function()
        vim.treesitter.start()
      end,
    })
    ```

    Alternatively, create individual ftplugin files (e.g., `~/.config/nvim/ftplugin/lua.lua`):

    ```lua
    vim.treesitter.start()
    ```
  </Step>

  <Step title="Verify installation">
    Check that everything is working:

    ```vim
    :checkhealth nvim-treesitter
    ```

    Open a file in one of your installed languages and verify that syntax highlighting is working correctly.
  </Step>
</Steps>

## Enable additional features

### Code folding

Enable tree-sitter-based code folding:

```lua
vim.api.nvim_create_autocmd('FileType', {
  pattern = { 'lua', 'python', 'javascript' },
  callback = function()
    vim.treesitter.start()
    vim.wo[0][0].foldexpr = 'v:lua.vim.treesitter.foldexpr()'
    vim.wo[0][0].foldmethod = 'expr'
  end,
})
```

### Indentation (experimental)

<Warning>
Tree-sitter-based indentation is experimental and may not work perfectly for all languages.
</Warning>

```lua
vim.api.nvim_create_autocmd('FileType', {
  pattern = { 'lua', 'python' },
  callback = function()
    vim.bo.indentexpr = "v:lua.require'nvim-treesitter'.indentexpr()"
  end,
})
```

## Common operations

### Update all parsers

Keep your parsers up to date with the latest versions:

```vim
:TSUpdate
```

### List installed parsers

See which parsers are currently installed:

```lua
:lua print(vim.inspect(require('nvim-treesitter').get_installed()))
```

### Uninstall a parser

Remove a parser you no longer need:

```vim
:TSUninstall rust
```
